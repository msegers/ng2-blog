Javascript 8+ Years

As many developers I met back in the day I started working, I used to hate Javascript, I felt it wasn't stable and might crash on different browsers, also there were loads of people disabling this in browsers. I ended up doing lots of work in libraries like ExtJs (Sencha) and started to feel more confident and attached to the language. Nowadays I tend to keep up to date about the development, that's why this website is written in Angular2 using TypeScript, a nicely typed language which at this time is being converted to regular Javascript using Traceur.
Libraries I spend time developing in: Prototype, Jquery, ExtJs, AngularJS, AngularJS 2. Besides these libraries I also worked a little with command line tools, like NodeJS, Grunt, Gulp, Bower etc.

Objective-C and Swift <1 Year

I'm not a big fan of Objective-C but I love Swift, however for getting around in iOS development you need basic knowledge of the language and API's it offers. During my graduation I had no help from other people with experience in these languages, so after some time working with Objective-C I did a small test how hard it would be to change the current code to swift. Without knowledge I wrote the 4 classes I had into swift within a few hours, and kept these like that. I love the Swiftness of development and forced type safety Swift enforces on the developer and working with the language felt like writing code in Google Go

Google Go (Home Experiments only)

When I first head of Google Go I had no idea what it was like, after a few months I started toying around with it since I needed to know! (I'm a huge Google Fanboy). I was especially amazed by how little code you had to write to run a webserver, or handling concurrency by simply writing "go" in front of a function. I experimented a little with Go, and actually have a program which would work as host for a blogging site which I want to get out there soon. A while back I applied for a Job which actually would allow me to become a Go Developer, but I ended up rejecting since I would mostly be doing Front-End stuff which was not worth the risk and travel. The program I wrote as a test however, is available on my Github Page and I feel happy sharing it with others.

As Person

I like to see myself as a Social and outgoing person, I tend to try and keep most of my team members smiling as we go but I could learn to give better commitments on deadlines avoiding dissapointment. I love learning new things and feel like I can do this in a rapid pace, and improving my skills in the stuff I know about and keeping up to date with developments. I can be quite easy to distract and I prefer to do things Ad-Hoc instead of planned out before me a little chaos can be found in most things I say and write but I try to make things clear. Because of my planning skills I like working in a Scrum based environment, besides being forced to work on smaller tasks, you plan once and go for the sprint making it easier to keep your planning in order. I prefer working in teams so I can learn from others and hopefully let other people learn from me as well, I get satisfaction out of helping others, but I do not ask for help that quickly.
